# AGENTS.md

Development guide for agents working on the Smart Network Learning Platform.

## Project Overview

This project is a React/Vite + Express + SQLite web application for a teacher portfolio, LMS content for course 31909-2005, 15 network competency games, learner dashboards, and admin reporting.

Core files:

- `src/main.jsx` contains the React app, learner UI, LMS screens, games, and admin panel.
- `src/styles.css` contains the Tailwind/CSS layer, command center UI, animations, and responsive rules.
- `server.js` contains the Express API, SQLite schema/seed logic, auth, uploads, reports, and static hosting.
- `data/lms-seed.json` contains initial LMS unit, lesson, and quiz data.
- `database.sqlite` is runtime data. Do not overwrite it during deployment unless explicitly requested.
- `dist/` is the production build generated by `npm run build`.
- `public/assets/` contains activity, LMS, profile, and supporting images.

## Common Commands

Install and run:

```powershell
npm install
npm run build
npm start
```

Frontend development:

```powershell
npm run dev
```

Check server syntax:

```powershell
node -c server.js
```

Check local health while the server is running:

```powershell
Invoke-RestMethod http://localhost:3000/api/health
```

## Safety Rules

- Do not delete or overwrite `database.sqlite` unless the user explicitly asks.
- Do not deploy `node_modules/` unless there is a clear operational reason.
- Before editing UI, inspect nearby patterns in `src/main.jsx` and `src/styles.css`.
- After frontend changes, run `npm run build`.
- After API/server changes, run `node -c server.js`.
- Do not commit or document passwords, tokens, or secrets.

## Current System Notes

- The app is expected to have 15 active games.
- LMS units should have `game.status = available`.
- The frontend and admin panel use a command center / cyber learning visual style.
- `AnimatedLearningMap` includes an animated bot/status layer for activity progress.
- `/api/game-data` should report `games = 15` and `units = 15`.

## Production Deployment

Production server:

- Path: `/opt/smart-network-learning`
- Service: `smart-network-learning.service`
- Internal port: `3000`
- Nginx reverse proxy target: `127.0.0.1:3000`

Back up before deployment:

```bash
ts=$(date +%Y%m%d-%H%M%S)
mkdir -p /root/backups
tar -czf /root/backups/smart-network-learning-$ts.tar.gz \
  -C /opt/smart-network-learning \
  --exclude=node_modules \
  --exclude=database.sqlite \
  --exclude=public/uploads \
  .
```

Usually deployed files:

- `server.js`
- `dist/`
- `src/` only to keep the latest source on the server

Post-deploy check:

```bash
chown -R www-data:www-data /opt/smart-network-learning/server.js /opt/smart-network-learning/dist /opt/smart-network-learning/src
systemctl restart smart-network-learning.service
systemctl status smart-network-learning.service --no-pager -l
curl -fsS http://127.0.0.1:3000/api/health
```

## Security

The production systemd service should set these environment values before real use:

- `ADMIN_USERNAME`
- `ADMIN_PASSWORD`
- `ADMIN_TOKEN`
- `PASSWORD_SALT`

If `/api/health` returns `needs-configuration`, a default value still needs to be changed.

## UI Guidance

- Preserve the command center / cyber learning visual direction.
- Prefer real images from `public/assets/` for activities and inspectable content.
- Keep animation CSS-driven where practical, and never let it block content readability.
- Learner pages should expose progress, lessons, games, quizzes, and activity logs clearly.
- Admin pages should expose learner progress, game activity, LMS reports, and config warnings.

## Graphify

This project has a knowledge graph at `graphify-out/` with god nodes, community structure, and cross-file relationships.

When the user types `/graphify`, invoke the `skill` tool with `skill: "graphify"` before doing anything else.

Rules:

- For codebase questions, first run `graphify query "<question>"` when `graphify-out/graph.json` exists.
- Use `graphify path "<A>" "<B>"` for relationships and `graphify explain "<concept>"` for focused concepts.
- Dirty `graphify-out/` files are expected after hooks or incremental updates; dirty graph files are not a reason to skip Graphify.
- Only skip Graphify if the task is about stale or incorrect graph output, or the user explicitly says not to use it.
- If `graphify-out/wiki/index.md` exists, use it for broad navigation instead of raw source browsing.
- Read `graphify-out/GRAPH_REPORT.md` only for broad architecture review or when query/path/explain do not surface enough context.
- After modifying code, run `graphify update .` to keep the graph current (AST-only, no API cost).
